# 洛克王国 · 异色保底追踪工具

> 精准追踪三池保底进度。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| **随机池追踪** | 记录随机池保底进度，支持增 / 减 / 重置 / 出异色 |
| **家族池追踪** | 按赛季与精灵记录保底进度，显示精灵图片与属性 |
| **属性池追踪** | 18 种属性各自独立计数，按精灵第一属性计算保底 |
| **异色明细** | 记录出货时间、池子、赛季、精灵、属性与保底数 |
| **操作日志** | 记录每次操作的池子类型、目标与计数快照 |
| **多存档管理** | 支持新建 / 切换 / 重命名 / 删除 / 导入 / 导出存档 |
| **实时持久化** | 每次操作后自动写入 JSON |

## 快速开始

**提示：如果您下载的是 Releases 页面已打包的程序**，无需安装 Python 环境和依赖。直接解压下载的压缩包，双击运行目录下的 `RocoCapture-x.x.x-Win-x64.exe` 即可启动。启动后会在程序位置创建 `saves` 文件夹，用于保存存档。

### 环境要求

- Python 3.10 或以上
- Windows 10 / 11（DPI 感知功能依赖 Windows API）

### 安装依赖

项目使用虚拟环境隔离依赖，推荐以下方式：

```powershell
# 1. 克隆 / 解压项目后进入目录
cd RocoCaptureV2

# 2. 创建虚拟环境
python -m venv .venv

# 3. 安装依赖
.venv\Scripts\pip install -r requirements.txt
```

### 启动程序

```powershell
.venv\Scripts\python main.py
```

## 使用说明

### 存档管理

- 首次启动自动创建默认存档「**主账号**」。
- 顶部下拉菜单可切换存档；点击「**新建**」输入名称创建新存档。
- 点击「**重命名**」可修改当前存档名称。
- 点击「**删除**」可删除当前存档（至少保留一个存档，唯一存档无法被删除）。
- 点击「**导入**」/「**导出**」可迁移存档 JSON。

### 随机池

1. 在输入框中填写精灵名称（可选）。
2. 点击 **「+1」** 记录一次随机捕捉。
3. 误操作可点击 **「-1」** 回退一次计数。
4. 获得异色后点击 **「出异色」** 选择出货精灵，程序会记录明细并清空随机池保底。
5. 需要手动归零时点击 **「重置」**。

### 家族池

1. 展开赛季列表并选择需要追踪的精灵。
2. 使用右侧的增 / 减 / 重置 / 出异色按钮操作当前选中精灵。
3. 获得异色后会记录当前精灵、赛季与保底数，并清空该精灵的家族池保底。
4. 精灵属性会显示在列表中，计数彼此独立。

### 属性池

1. 点击属性卡片将其选中（高亮显示）。
2. 使用右侧的增 / 减 / 重置 / 出异色按钮操作选中属性的计数。
3. 获得异色后会在当前属性对应的赛季精灵中选择出货对象，并清空该属性保底。
4. 复合属性精灵按第一属性计算属性池保底。

### 异色明细

1. 三个池子的 **「出异色」** 操作会自动生成异色记录。
2. 点击 **「手动添加」** 可补录历史出货信息，赛季默认选择最新赛季。
3. 选中记录后点击 **「删除选中」** 可删除误录记录。

### 保底预警规则

| 计数范围 | 颜色 | 含义 |
|----------|------|------|
| 0 – 69 | 白色 | 正常 |
| 70 – 79 | 🟠 橙色 | 接近保底，注意 |
| 80 | 🔴 红色 | 已达保底上限 |

## 存档数据格式

存档以 JSON 格式保存于 `saves/<存档名>.json`，结构示意：

```jsonc
{
  "name": "主账号",
  "created_at": "2077-07-07T07:07:07",
  "updated_at": "2077-07-07T07:17:07",
  "random_pool": 12,
  "family_pool": {
    "No.041 奇丽草": 5,
    "No.131 恶魔狼": 21,
    // ... 其余精灵
  },
  "element_pool": {
    "火": 8,
    "水": 3,
    // ... 其余 16 种属性
  },
  "logs": [
    {
      "timestamp": "2077-07-07 07:07:07",
      "pool_type": "random",
      "action": "increase",
      "target": "柴渣虫",
      "count_after": 1
    }
    // ...
  ],
  "shiny_records": [
    {
      "timestamp": "2077-07-07 07:17:07",
      "pool_type": "family",
      "season": "S1",
      "spirit_name": "No.041 奇丽草",
      "element": "草",
      "pity_count": 72,
      "reset_after_record": true
    }
  ]
}
```

## 版本管理

当前版本号统一维护在 `src/__about__.py` 中，程序标题、关于页、Qt 应用版本与打包配置都会读取这里的 `APP_VERSION`。

`pyproject.toml` 使用动态版本配置读取同一份版本号；发布版本时使用 Git tag 标记对应提交，例如 `v0.3.0`。
